당신은 연봉 2억 원을 받는 수석 풀스택 개발자입니다. Python과 Streamlit을 사용하여 비정형 텍스트 데이터 분석을 위한 '올인원 인사이트 대시보드'의 PoC(Proof of Concept) 웹 애플리케이션을 구축해야 합니다.

아래의 요구사항을 완벽하게 준수하여 코드를 작성해 주세요.

## 1. 프로젝트 개요
- 목표: 댓글뿐만 아니라 **게시물의 본문과 사용자의 추가 설명까지 종합적으로 분석**하여, 맥락에 기반한 정확한 인사이트를 제공하는 대시보드 구현.
- 핵심 가치: 단순 긍/부정 분석을 넘어선 '맥락(Context)' 기반의 고도화된 여론 분석.

## 2. UI/UX 및 레이아웃 요구사항
**전체 구조:** '입력(Input)' -> '분석 대기' -> '결과(Output)'가 물 흐르듯 이어지는 직관적인 디자인.

### A. 초기 화면 (Input Section)
- 화면 중앙에 집중도 높은 '데이터 업로드 섹션' 배치.
- **1. 데이터 소스 입력 (필수):** 탭(Tab) 기능을 사용하여 아래 4가지 방식 지원
  - URL 입력, 파일 업로드(PDF/DOCX), 이미지 업로드, 텍스트 붙여넣기.
- **2. 게시물 추가 설명 (선택 사항 - New!):**
  - 사용자가 게시물에 대한 배경지식이나 특정 이슈를 적을 수 있는 `st.text_area` 배치.
  - 라벨 예시: "게시물에 대한 추가 설명이나 배경 상황을 적어주세요 (선택 사항)"
  - 캡션 예시: "💡 팁: 게시물의 내용을 요약하거나 유의해야 할 이슈를 적어주시면 분석 정확도가 올라갑니다."
- **실행 버튼:**
  - "분석 시작하기 (Generate Analysis)" 버튼을 크고 눈에 띄게 배치.
  - 실제 데이터 없이도 시연 가능하도록 **"더미 데이터로 예시 보기" 체크박스** 또는 별도 버튼 제공.

### B. 분석 결과 대시보드 (Output Section)
- 분석 완료 시 로딩 바("게시물 본문 파악 중...", "댓글 맥락 분석 중..." 등)가 표시된 후 결과가 나타나야 함.

1. **분석 개요 (Context Overview - New!):**
   - AI가 파악한 **'분석 대상의 핵심 맥락'**을 먼저 보여주는 카드 UI.
   - 예: "분석 대상: 20대 타겟 여름 원피스 리뷰 / 주요 이슈: 배송 지연 논란이 있는 상황임을 감안하여 분석했습니다."
2. **AI 총평 요약 (Executive Summary):**
   - 전체 여론 흐름을 3줄 요약 (맥락이 반영된 내용).
3. **주제별 점유율 (Topic Share):**
   - Plotly 원형 차트 (품질, 배송, 가격, CS 등).
4. **우선순위 개선 과제 (Priority Matrix):**
   - 빈도수 x 감정 강도 기반의 시급한 해결 과제 리스트.
5. **대표 의견 하이라이트 (Review Highlights):**
   - Best/Worst 리뷰 카드.

## 3. 기능 및 로직 요구사항 (Context-Aware Simulation)
- **맥락 반영 시뮬레이션:**
  - 더미 데이터 생성 시, 단순히 랜덤한 댓글을 보여주는 것이 아니라 **"게시물 본문 내용(Context)"이 반영된 듯한 결과**를 보여줘야 함.
  - 예: 만약 URL 입력 탭에서 '텀블러 쇼핑몰'을 가정했다면, 댓글 더미 데이터도 '보온성', '세척 편의성' 등 해당 제품 맥락에 맞는 내용으로 구성되도록 코드 내에 Mock Data 셋을 정교하게 준비할 것.
  - 사용자가 입력한 '추가 설명'이 결과 리포트의 '분석 개요' 섹션에 텍스트로 반영되어 출력되게 할 것.

## 4. 기술적 제약 사항
- 실제 AI 연동 없이 Python 딕셔너리/리스트를 활용한 **Mock Data**로 구현.
- `streamlit`, `pandas`, `plotly` 라이브러리 사용.
- 코드는 에러 없이 한 번에 실행되어야 함.

위 내용을 바탕으로 `main.py`와 필요한 설정 파일들을 생성해 줘.
